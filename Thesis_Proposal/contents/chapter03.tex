\chapter{研究方法}

本研究旨在開發一套整合光機電系統之成本效益高且具備高度擴充性的樂高螢光倒立顯微鏡。為了克服低成本光學元件在物理極限上的先天限制，本系統採取了軟硬體高度協同整合的設計哲學。整體研究方法涵蓋了硬體機構的光機設計、底層嵌入式韌體的精確控制、基於網頁架構的高階多執行緒軟體系統，以及導入計算成像概念的先進影像處理演算法。以下將依序深入闡述各個子系統之設計原理、架構規劃與實作細節。

\section{顯微鏡硬體架構}
傳統正立顯微鏡在載台掃描範圍與樣本容器高度上往往面臨嚴格的實體干涉限制。為此，本研究採用了倒立式光路配置，將高數值孔徑的物鏡模組精確倒置於掃描載台之下。在螢光光路設計中，特定波長的激發光源首先穿透激發濾波片，經由以四十五度角放置的二向色鏡反射後，由下而上穿透物鏡並聚焦於樣本平面。樣本受激發後所放射出的較長波長螢光訊號，則循原光路向下穿透二向色鏡與發射濾光片，最終由成像透鏡匯聚於互補式金屬氧化物半導體（CMOS）感測器上完成光電轉換。在機械結構方面，為了同時兼顧製造成本與系統擴充性，機身主體骨架大量採用樂高積木建構。樂高積木在射出成型時具備極高的幾何網格精度，本研究利用此特性作為光學同軸對齊的初步基準，並輔以客製化設計的三維列印轉接零件來夾持並精確鎖固各項光學元件，從而打造出一個高剛性且可自由重構的光學平台。載台的空間運動導入了基於笛卡爾座標系統的剛性龍門架構，透過三顆高精度步進馬達驅動。其中 X 軸與 Y 軸負責樣本水平面上的大範圍、高通量掃描，而 Z 軸則專司物鏡垂直方向上的微米級升降，以執行嚴格的光學對焦任務。

\section{軟體設計}
為了實現跨設備的遠端操作彈性與高度自動化的實驗流程，本系統的主控軟體揚棄了傳統單機版視窗程式的開發模式，轉而採用前後端分離的現代化網路架構。系統後端核心採用Python 3程式語言進行開發，並選用 PySide6 框架來進行底層資源的調度與事件管理。為了讓前端網頁介面能夠透過標準的超文本傳輸協定（HTTP）與硬體進行即時通訊，本研究利用Python內建的http.server與socketserver模組，從底層建構了一套支援多執行緒並發處理的輕量級網頁伺服器。該伺服器具備自定義的應用程式介面（API）解析能力，負責將使用者的網頁點擊行為轉化為顯微鏡的物理動作。

\subsection{多執行緒任務管理}
顯微鏡的自動化操作本質上涉及大量高耗時與計算密集的任務，例如百萬像素級別的即時影像矩陣運算、多階段自動對焦演算法的執行，以及大範圍螢光影像的自動化採集等。若將這些任務全數交由伺服器的主執行緒循序處理，極易導致系統阻塞，進而造成網頁控制介面失去響應（Freeze）。為徹底解決此一同步阻塞問題，本研究導入了嚴謹的工作執行緒模型（Worker Thread Model）。系統主程式專職負責維護 HTTP 伺服器的連線監聽與事件循環（Event Loop），而將連續影像擷取、色彩空間轉換等高負載工作封裝於獨立的 CameraWorker 類別中。同理，自動對焦與 Z 軸深度疊合掃描（Z-Stack）亦由專屬的工作類別接管。透過 Qt 框架提供的 moveToThread 機制，這些工作類別被精確地派發至獨立的系統執行緒中平行運作，並被賦予最高的執行優先權，確保硬體中斷與影像處理演算法不受 I/O 延遲干擾，大幅提升了系統在長時間連續運行下的極限穩定度。

\subsection{模組化硬體抽象層}
為確保系統軟體具備良好的可維護性與硬體可移植性，本研究於軟體架構中實作了一層硬體抽象層（Hardware Abstraction Layer），將所有與底層實體裝置溝通的邏輯深度封裝於獨立的模組中。具體而言，明場與螢光激發光源的數位腳位切換由 LightControl 類別全權處理，而三軸步進馬達的相位轉換則交由 MotorControl 類別負責。此抽象層的核心價值在於解耦了「物理位移」與「馬達控制指令」之間的關聯：當上層應用程式發出一個以微米（\textmu m）為單位的移動請求時，該模組會自動查閱系統配置設定檔，套用對應的導螺桿導程與微步（Micro-stepping）轉換率，將微米數精準換算為步進馬達的脈衝步數，隨後打包成自定義的序列通訊指令（Serial Protocol）發送至微控制器。這種設計使得未來若需替換不同規格的導螺桿或感測器，僅需修改配置模組即可，無需重構任何核心控制邏輯。

\subsection{資料流程與即時監控}
在動態影像擷取與串流傳輸的資料流設計上，本系統採用了經典的生產者-消費者模式（Producer-Consumer Pattern）來調節資料吞吐量差異。運行於背景獨立執行緒中的影像擷取模組扮演生產者角色，負責連續觸發相機感測器，並在第一時間對原始影像矩陣執行平場校正與白平衡等基礎預處理。處理完畢的影像陣列會被安全地寫入具備執行緒同步鎖定機制（Mutex Locks）的緩衝區中。另一方面，網頁伺服器的串流處理常式則作為消費者，不斷從該緩衝區提取最新影像，將其即時編碼為 Motion JPEG（MJPEG）格式，並透過 HTTP 分塊傳輸編碼（Chunked Transfer Encoding）無縫推播至使用者的瀏覽器端。為進一步避免儲存裝置的寫入瓶頸拖慢即時預覽的幀率，系統亦建置了專屬的非同步 I/O 任務佇列，將硬碟存檔與影片編碼等高度耗時的操作轉移至背景默默執行。

\section{韌體架構}
韌體部分運行於 Arduino 微控制器上，負責解析來自上位機的指令並驅動三軸步進馬達。為了減輕馬達啟停瞬間的震動並防止高速運動下產生失步，系統利用了 AccelStepper 函式庫，實現了具備平滑加減速路徑規劃（加速度高達 160,000 步/秒\textsuperscript{2}）的運動控制。在通訊層面，韌體實作了一套類似 G-code 的 ASCII 格式序列埠指令解析協定。該協定能即時解析如 \texttt{x50}（X軸位移50步）、\texttt{z-20}（Z軸反向位移20步）等字串格式，將其轉化為相對移動的脈衝訊號。

此外，韌體導入了具備距離感知的動態速度規劃機制。系統在接收指令後會擷取步數的絕對值，自動評估並切換適用的速度配置：當移動距離超過 1000 步之閾值時，系統會切換至高速模式（16,000 步/秒），藉此優化大範圍掃描與歸零操作的時間效率；而在 1000 步以內的短距離移動或自動對焦過程中，則維持常規的最大速度（800 步/秒），以確保定位的極致精確度並有效抑制載台過衝之現象。

在運動執行面上，韌體採用了阻塞型（Blocking）的 \texttt{runToPosition()} 函數，確保馬達在每次相對移動完成之前處於等待狀態，以保證空間定位的絕對穩定度。同時，韌體架構中亦內建了各軸向運動方向的自定義功能（如設定 Y 軸與 Z 軸反轉），賦予系統高度的相容性，以彈性適應各類機構改裝與組裝需求。

\section{影像處理演算法}
受限於消費級感光元件的半導體製程限制與三維列印機構的公差，本系統在原始光學成像上必然存在熱雜訊、鏡頭陰影（Lens Shading）以及色彩串擾等物理缺陷。為了將低成本硬體擷取到的原始數據提升至具備科學定量分析價值的標準，本研究開發並實作了一套嚴謹且多層次的影像運算處理管線（Image Processing Pipeline）。該演算法管線自最底層的物理缺陷補償起步，向上延伸至多階段自動對焦，接著透過影像拼接技術擴展空間視場，最終利用傅立葉疊層演算法在頻域中突破硬體的繞射極限。

\subsection{影像校正與預處理}
感測器獲取的原始信號在進行任何高階運算前，必須先經過嚴格的物理基準校正。首要程序為黑電平校正（Black Level Correction, BLC）。在完全無光的理想環境下，CMOS 感測器受限於半導體暗電流與熱雜訊的影響，其像素讀數仍會呈現非零的基線電壓。為確保影像訊號具備絕對的線性物理意義，系統在解析 Bayer RAW 陣列時，會強制扣除特定感測器（如 Sony IMX219）固有的暗電流偏移數值，確保絕對黑體在數位影像中精確對應為數值零，避免底噪在後續增益運算中被等比例放大。 

緊接著，系統針對光學系統產生的漸暈（Vignetting）與亮度分佈不均執行平場校正（Flat-field Correction）。演算法會預先提取一張均勻明場影像作為基準，計算影像幾何中心區域亮度的第九十五百分位數作為全圖的理想標竿。隨後，演算法逐像素計算全圖亮度與該標竿的相對衰減比例，藉此生成紅、綠、藍三通道獨立的空間增益矩陣（Gain Maps）。在每次成像時，原始影像將與該矩陣進行純量乘法，以數位方式將邊緣衰減的亮度完美補償回中心水準。在微弱的螢光成像模式下，系統更額外導入了動態零點校準（Auto-Zeroing）機制，透過動態抽樣全圖最暗的第一百分位數值作為即時底噪基準並予以全域扣除，藉此大幅推升螢光主體的訊噪比。 

針對微透鏡陣列因主光線角度（Chief Ray Angle）不匹配所引發的邊緣像素色彩串擾（Color Crosstalk），本研究引進了先進的色彩解混（Color Unmixing）技術。該演算法利用一組高維度的空間變異張量（Spatially Varying Tensor），並透過愛因斯坦求和約定（Einstein Summation）進行高效率的矩陣運算，將每一個空間座標上的失真 RGB 數值，透過其專屬的反向變換矩陣重新映射回真實的光譜空間，精確消除螢光影像在視場邊緣發白或偏色的物理失真。

\subsection{自動對焦}
為了解決自動化掃描過程中因培養皿表面微小起伏所造成的離焦現象，本研究在 Fu 等人提出的改進型爬山搜尋演算法（Mountain-Climb Searching, MCS）基礎上，針對螢光影像極端低對比與高雜訊的環境進行了深度客製化。本系統的自動對焦邏輯採取多階段迭代逼近策略，預先定義了由大至小（如 100、30、10、3、1 步）的遞減步進序列，在搜索速度與最終收斂精度之間取得完美平衡。

在演算法的評估函數（Fitness Function）選擇上，系統導入了動態切換機制。於粗調大步距階段，演算法採用計算開銷極低且全域單峰性良好的影像變異數（Variance）函數，以最快速度判斷焦平面的大略走向。當搜索進入最後兩階的精微逼近時，明場模式將無縫切換至對高頻邊緣極度敏感的 Tenengrad 梯度函數；而針對背景常充滿雜訊的螢光模式，本研究開發了專屬的提取邏輯：影像將先經過大尺度的形態學開運算（Morphological Opening）徹底抹除背景散斑，隨後僅提取影像梯度直方圖中前 0.1\% 的極高強度像素來計算均方根值，強制演算法將焦點鎖定於真實發光的細胞結構上。

在搜尋的空間導航邏輯上，系統具備極高的容錯與自適應能力。在演算法啟動的首步移動後，若清晰度函數出現下降，系統將立即判定初始方向錯誤，不僅強制反轉搜尋向量，更會啟動兩倍步距補償機制，以確保載台能瞬間脫離錯誤的離焦軌道。此外，為防止在大步距掃描時直接「跨越山頭」而遺漏極值點，演算法持續監控清晰度跌落幅度；一旦數值跌穿歷史最大值的百分之七十，系統便認定已發生嚴重過衝，將立刻截斷當前迴圈並退回上一步，啟動下一階段的精細掃描。唯有當系統連續兩次偵測到清晰度微幅下降時，方才在數學上嚴格確立已越過真正的光學峰值，隨後系統將精準反推馬達步數，將物鏡完美停泊於最銳利的焦平面位置。

\subsection{影像拼接}
針對高倍率物鏡所導致的狹小視場限制，本系統實作了基於像素強度（Intensity-based）的影像配準與拼接演算法。相較於依賴特徵點提取的演算法在面對稀疏螢光樣本時容易失效，本研究採用正規化互相關（Normalized Cross Correlation, NCC）作為兩張相鄰影像間空間重疊相似度的數學評估指標。在處理水平相鄰的影像對（Pair-wise）時，演算法會模擬二維空間的滑動視窗，透過多階段的位移匹配策略——從大範圍的粗略平移估算，逐步縮小至局部誤差區域的亞像素級精細比對——來尋找使得 NCC 數值最大化的全域最佳解。

當面對多孔盤中數以百計的龐大影像陣列時，系統採取了全局最佳化的拓樸策略。演算法將所有相鄰影像配對的 NCC 數值倒數轉化為圖論中的邊權重，並運用最小生成樹（Minimum Spanning Tree, MST）演算法來規劃全域影像的最佳合成路徑。此種基於圖論的拓樸降階技術，能夠有效避免單一不良拼接配對造成的誤差在整個矩陣中呈現雪崩式的累積擴散，從而確保大範圍螢光拼接影像的幾何精確度與視覺平滑性。

\subsection{螢光傅立葉疊層}
為了在不升級昂貴高數值孔徑物鏡的前提下獲取奈米級的超解析度結構資訊，本研究突破性地在低成本平台上導入了圖樣化照明螢光傅立葉疊層（Pattern-Illuminated Fourier Ptychography, piFPM）演算法。該演算法的核心物理機制在於：透過在激發光源路徑中置入帶有隨機散斑的擴散片，人為地改變樣本表面的螢光激發強度分佈，藉由空間頻率混合（Frequency Mixing）效應，將原本隱藏於光學系統截止頻率之外的超高頻細節，調變並折疊回物鏡可觀測的通帶範圍內。

演算法的實際執行分為兩大數學階段。第一階段為嚴謹的空間序列配準。考量到機械公差造成的位移誤差會徹底破壞疊層演算法的收斂性，系統首先利用高斯模糊與對數轉換（Log Transformation）對原始影像進行特徵強化與動態範圍壓縮，接著應用頻域相位相關法（Phase Correlation），計算各影格相對於基準影像的互功率譜，以獲取極高精度的亞像素級位移向量矩陣。

進入第二階段的疊層重建後，演算法採用交替投影（Alternating Projection）的優化策略。在每一次的數學迭代中，演算法會根據系統的物理參數（如激發波長與數值孔徑）建立非相干光學轉換函數（Incoherent OTF）作為低通約束條件。隨後，將當前估測的高解析度樣本頻譜與經空間位移對齊的照明圖案在空間域進行點積運算，模擬螢光激發的物理過程，並計算其與實際觀測影像之間的強度殘差。最後，運用梯度下降法（Gradient Descent）於頻域中反覆雙向更新樣本結構與照明圖案的估測矩陣。經過數十次的迭代運算後，高頻資訊將被逐步解調並正確歸位於廣闊的頻譜空間中，最終實現超越感測器原生繞射極限的超解析度螢光顯微成像。
